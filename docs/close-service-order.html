Payment Boundary

GitHub issues, page views, generated scripts, and buyer conversations count as $0. Paid work starts only after scope is accepted and payment is made through a seller-controlled external route such as a checkout, invoice, marketplace order, funded milestone, posted transfer, or other saveable proof.

Fast Close Sequence

  1. Pick the exact service row that matches the buyer's problem.
  2. Send the buyer reply script and request only safe, public, or buyer-authorized inputs.
  3. Paste the scope acceptance script into your controlled channel after the buyer confirms scope.
  4. Create the external payment request using the invoice line item.
  5. Start work only after payment/proof is funded, posted, released, payable, or otherwise saveable.
  6. Deliver the listed artifacts, save proof, and count only verified money after holds/refunds.
